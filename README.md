
# SOE Software Store — متجر البرمجيات

## التشغيل المحلي
1. ثبّت Node.js 18 أو أحدث.
2. شغّل:
   ```bash
   npm install
   npm start
   ```
3. افتح:
   `http://localhost:3000`

القاعدة ستُنشأ تلقائياً داخل:
`data/soe.sqlite`

بيانات المدير الافتراضية:
`admin@soe.com` / `Admin123!`

## النشر المجاني
أفضل خيار مجاني مناسب لهذا المشروع هو **Koyeb** لأن الخطة المجانية تدعم تطبيقات Node.js، بينما Render في الخطة المجانية لا يحفظ تعديلات نظام الملفات المحلية، وهو غير مناسب لقاعدة SQLite المحلية إلا مع تخزين دائم مدفوع. Koyeb يوفّر خطة مجانية، ويدعم أيضاً التخزين الدائم عبر Volumes لكن هذه الميزة موضوعة في معاينة عامة ويجب الانتباه إليها. citeturn627798search2turn627798search3turn627798search15

### خطوات النشر على Koyeb
1. ارفع المشروع إلى GitHub.
2. افتح حساب Koyeb.
3. أنشئ Service جديدة من GitHub repo.
4. اختر Node.js build و start command:
   ```bash
   npm start
   ```
5. أضف المتغيرات التالية:
   ```env
   NODE_ENV=production
   JWT_SECRET=اكتب_نص_طويل_عشوائي
   SQLITE_PATH=/data/soe.sqlite
   ```
6. بعد التشغيل افتح رابط الخدمة.

### لو أردت بيانات تبقى بعد إعادة النشر
أنشئ Volume واربطه بمسار:
`/data`

بهذا يبقى ملف `soe.sqlite` ثابتاً بين عمليات إعادة النشر. ميزات التخزين الدائم على Koyeb موجودة، لكن توثيقهم يشير إلى أن الـ Volumes ما زالت في public preview ومناسبة للاختبار أكثر من الاعتماد الحرج. citeturn627798search3turn627798search8
